Setting Up your Dev Environment

Aside from owning a computer and having access to the internet you will also need:

  1. an IDE text-editor to program your application
  2. a Local Host-Server to work on a project in real-time offline
  3. a Version Control System to manage your work

An Integrated Development Environment is a software application that provides comprehensive facilities to computer programmers for software development. IDEs are designed to maximize programmer productivity by providing versitle components in a single program by which all development is done. In general, an IDE is a graphical user interface (GUI)-based workbench designed to aid a developer in building software applications with an integrated source-code editor combined with all the required (automation) tools at hand. This program typically provides many features for authoring, modifying, compiling, deploying and debugging software, which helps developers quickly execute actions without switching to other applications.[1] My IDE of choice is Brackets. It's really easy to use so I recommend checking it out.[2]

A Local Host-Server is a computer networking configuration that creates an online-like environment for developers to test applications without internet access. Specifically, localhost is the host name for your computer and uses the IP address 127.0.0.1 to act as a loopback mechanism used to run a network service on a host without requiring a physical network interface.[3] Configuring a local host manually is beyond the scope of this project so I strongly suggest that you head over to Apache Friends and download their open source cross-platform web server-stack.[4] Note: Before downloading this program, research alternatives that are best suited for you. For example, I work within a Microsoft Windows Operating System and there is a WAMP server-stack designed specifically for Microsoft Windows OS. However, my preferred server-stack is XAMPP due to its versitility.

A Version Control System is a software configuration that records changes to web files over time so that you can recall specific versions later. For all intents and purposes, a VCS is an online storage facility for your code, enabling dDevelopers with the ability to manage source-code, track all file versions (independent of network access) and coordinate a non-linear workflow in a team environment.[5] Github is my VCS of choice. Register to get started.[6]

What is HTML?

HTML is a markup language that defines the structure of the content on a webpage. Sections of markup are defined with Elements, which are used to carve out sections of the structure, HTML Tags are used to define those elements and Internet browsers read and displays the HTML content to the user. Tags have an opening and closing component, with a few exceptions. Moreover, we can add Attributes to a tag to specify how the content will display (explained later with CSS).[7] Although this section provides only basic principles required of an HTML template, you can vist them good folks over at Mozilla to learn everything there is to know about web design and development. Better yet, check out the free HTML and CSS tutorial Shay Howe put together: its perfect for an absolute beginner!

Required Elements for an HTML Template

Moving on, the structural requirements for an HTML template are as follow:

The <!DOCTYPE html> is the first line of a page. I do not know why.

The <html> element (aka 'root') is on the second and final line of a page. This element is a wrapper for everything on the entire page: that which can be seen or not!

The <head> element is the area consisting of that which you cannot see and is used to store pertinent data on a page. Within this area you can use <meta> <title> and <link> tags to make the page mobile-compatible, document the webpage in the browser and specify resources being used, respectively.

The final required element is the <body> and is the area containing all content that you want displayed to users.

Defining Sections and Headings

Inside of the body is where all the magic happens. It is here that you can outline information that you want displayed based on the type of information being offered. The most important HTML elements that you can use to section information are the Header, Nav, Main, Section, Article, Aside and Footer.[8]

The <header> tag, when used, is the first section users encounter and contains vital information related to the website such as a logo, author or navigational aids. The <nav> tag can be used within the header or independently to help users navigate a webpage. A nav tag can provide links within the current document or to other documents with common examples including menus, tables of contents and indexes.

At the bottom of the page, before the closing <body> tag, you can use the <footer> tag to highlight the page author, copyright data or social media links.

The <main> and <section> tags can be a bit confusing because although they are semantically different (the <main> is actually a text content element), act nearly the same when used in the same situation. What is important here is to use the <section> tag as the main container for a section and use the <main> tag for the most important content of the body.

The <article> tag is used to contain content intended to be independently distributabble such as a magazine article or blog post.

The <aside> tag is used for content that is indrectly related to the documents main content.

Defining Content

Content elements are used to organize sections of content and are nested within any section element. There are dozens of elements that can be used to markup your content, but choosing the right element depends wholly on that which you are marking up. Aside from adding identifying sections of content with headings and adding text, you may want to include an list, a form, some sort of multimedia, a script, interactivity or a web component. There are dozens of elements at your disposal, but the only way you will understand them is to study them.[9] [10] [11]

In the example above, there are four kinds of tags used, all of which no project goes without.

The <h1> and <h4> tags illustrate a Heading that is used to define the title of any section. There are six heading levels <h1> - <h6> which decrease in size from 1 to 6.

The <p> tag represents a paragraph of text. The <a> tag is an inline-text semantic creates a hyperlink to another webpage, file, or location within the same page. This is an inline text semantic which helps define the meaning or purpose of a word, line or text.

The <ul> and <li> tags are used in unison to create an unordered list of items, which will display as bullet points. You could also use the <ol> tag to create an ordered list, which will display an ascending counter.

Clarification on how to define content can be found in this handy HTML Basics tutorial. It is important to acknowledge Attributes and how they can be used to manipulate the state of a particular element. An example is illustrated in the <img> tag where <img src="..." references the File Source of an image. Wait for the CSS section to learn more.
