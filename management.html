usb Version Control & Using Github

Git is a distributed version control system that helps developers control the different versions of the files in a project. In a distributed system, you can copy a complete repository with the full project history to every developer's machine.

Developers utilize Git via commands are delivered via console that can access the collection of older versions of project's which is called a repository. You can edit your files, then run Git commands to store your changes. Each time you complete a change to a project file, Git records a snapshot of their current contents. These snapshots are known as commits.

List of Git Commands and Descriptions

git Command git Description
git --help Displays a List of Git Commands
git clone Setup a New Repository
git init Setup a New Repository
git status Displays List regarding Status of Files
git log (-p) Displays a List of Old Commits (Show Lines Added to each File)
git ls Displays a List of all Repo Files
git add (.) Used to Stage a File (All Files)
git commit(-a -m) Used to Commit a File (All Files)
git push Used to Synchronize Commits with Remote Repositories
git pull Used to Synchronize Commits with Remote Repositories
git mv Move Files that are Tracked by Git
git rm Remove Files that are Tracked by Git

Git Commands include 'Options' that can be specified with a Single dash (-) followed by a Letter or a Double dash (--) followed by a Word.

Creating a Repository

There are two ways to store a Repository onto your computer. You can either Clone or copy a repo from another machine, or you can Create a new repo via Initializing.

Step 1 Initialize a Repo git init
Step 2 Stage Files git add filename.extension
Step 3 Commit File Changes git commit -m "Description of Commit"
Commits should follow the line, 'This commit will....'
Step 4 Input User Credentials git config --global user.email "principal@mistermoody.com"
(This is a One-Time Process) git config --global user.name "Mister Moody"
Step 5 Retrieve Remote Repo git remote add origin https://github.com/MisterMoody/projectName.git
git push -u origin master

The git log command produces a list of Commits along with their Simple Hashing Algorithm (SHM) identification number, which is a checksum of all changes in that commit. he checksum is the result of combining all the changes in the commit and feeding them to an algorithm that generates these 40-character strings. When you need to select a commit from your history, you can use these SHA-1 checksums, or "SHAs" for short, to identify which commit you want.

Working with Remote Repositories

Using github for independent projects is slightly different when working with others. For starters, a repo that you own is called a local repo whilst a repo owned by another developer is called a remote repo. Each repos is a full copy of its original, including version history.

While developers can make commits to their own repo independent of others, a dev must always pull commits other developers have made from a remote repo to a local repo to maintain the most up-to-date version.

Collaborators typically use a declared central repository and gain acces via cloning into their local machine. This allows developers to work on different components that can be added to the central repo by pushing commits. Simple add the repo by name to the folder on your local machine.

Clone a Repository git clone https://github.com/user/repo newlocalFolder
Update Repo git pull
Always Update Repo each time you Begin New Work
View Remote Repos git remote
'origin' is the Default Name for Remote Repos; they can be renamed.
Pull Changes git pull origin
This will add updated files to your working directory.
Add Remote Repos git remote add origin https://github.ProjectURL.git
git push origin master master
