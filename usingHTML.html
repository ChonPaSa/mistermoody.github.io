What is HTML?

HTML is a markup language that defines the structure of the content on a webpage. Every HTML document is a hierarchical structure made up of Elements that dictate how content is displayed and/or sectioned. HTML Elements, also referred to as Tags, are used by Internet browsers to read and display HTML content to a user. Elements usually have an opening <element> and closing </element> component, but there are a few exceptions in which there is no closing component.

Moreover, element content can be manipulated through the use of Attributes, which are associated properties that are used to extend the capability of an element, a configuration of sorts to meet certain criteria. An attribute is placed in the opening tag and looks something like attr="..." where the attribute name is followed by a delimiter and its actual value. There are more than 100 attributes that can be paired with elements depending on the objective, but there are two that I use often: class"..." and id"...", both of which deal more with CSS than HTML so I will delay further explanation until then. For now, I would like to focus on all the requirements for developing an HTML template.

Required Elements for an HTML Template

When creating an HTML template, I prepare content for four distinctive areas: the Main Root, Metadata, Section Root and Content Sectioning/Text Content.

The Main Root

The <!DOCTYPE html> is the first line of a page; it declares that this is an HTML file. The <html> element is on the second and final line of a page. This element is the outermost container that acts as a wrapper for everything on the entire page. There are only two direct descendants of this element and they are explained below.

Metadata

The second distinctive area and first direct descendant of the <html> element is the <head> element, which is the area that contains all meta information about a particular page, instructing the browser where to find style sheets while defining relationships that the document has to others in the web site. And although outside of my scope at this juncture, it is worth nothing that scripts that the document will need to apply later on can be added here as well.

The first line in this area is the <meta> element, which is a self-closing tag used to provide meta data about a webpage, its author, etc. Making a website mobile compatible, for example, is one way to use this tag and will require three attributes:

charset="utf-8"
Declares the character encoding of the page or script.
name="viewport"
A value used by the server for identifying its usage.
content="width=device-width, initial-scale=1.0"
A value associated with name (or http-equiv).

The second line in this area is the <title> element, which is used to document the webpage in the browser. Whatever is placed here will be visible on the browser tab. The third line in this area is the <link> element, which specifies the resources incorporated into a website and will always contain two specific attributes:

rel="stylesheet"
Specifies the relationship to the link object.
href="stylesheet"
The URL of a linked resource (internal or external).

Section Root

The third distinctive area and second direct descendant of the <html> element is the <body> element, which wraps all content that will be displayed on screen (or in other media, such as print), such as headings, paragraphs, images, tables, and so on. Everything that is within this element comprises what I consider to be the final distinctive area that is used to section content and create contextual elements as aforementioned.

Before moving forward, I want to point out the usage of Comments <!-- --> and that these are used in HTML for developers to make notes about what they are doing. Comments are not displayed when the HTML document is rendered by a browser so any content contained within them will be hidden!

Defining Sections

During the planning phase, I often imagine how I want the content to be displayed physically and how that content should behave. Doing so helps me determine which HTML element to use. When defining page sections, I know that I can rely on these elements to get me started: Header, Nav, Article, Section, Main, Aside, Div and Footer.

The <header> element is the first section users encounter and contains introdutory information related to the website such as a logo, author or navigational aids.

The <nav> element allows users to navigate a website, providing links within the current document or to an external source. This element can be used within a <header> or independently, and should be reserved for primary navigation sections only including global navigation, a table of contents, previous/next links, etc.

The <article> element represents a self-contained composition in a document that is intended to be distributable such as a forum post, a newspaper article, or a blog entry. This is 1 of 4 elements that I use to section main areas of content. As a superior element, it cannot be nested inside of any other element.

The <section> element represents a standalone section — which doesn't have a specific semantic element to represent it — contained within an HTML document. It can be nested inside of the <article> element to create distinguishable sections.

The <aside> element is used for content that is indrectly related to the documents main content such as sidebars, inserts, or brief explanations. These elements are frequently presented as sidebars or call-out boxes. When used within an <article> element, for example, the <aside> tag may identify content related to the author of the article.

The <div> element is a generic block-level element that doesn’t convey any meaning about its contents, but is the most common method for identifying the structural sections of a document and for laying out a web page using CSS. I use this element in the same fashion that I described the <main>. The <footer> element typically contains information about the author of the section, copyright data or links to related documents. It is placed at the bottom of the section root or used at the bottom of any sectioning elements.

Defining Content

Content elements are used to organize sections of content and are nested within any section element. There are dozens of elements that can be used to markup your content, but choosing the right element depends wholly on that which you are marking up. Aside from adding identifying sections of content with headings and adding text, you may want to include an list, a form, some sort of multimedia, a script, interactivity or a web component. There are dozens of elements at your disposal, but the only way you will understand them is to study them.[9] [10] [11]

In the example above, there are four kinds of tags used, all of which no project goes without.

The <h1> and <h4> tags illustrate a Heading that is used to define the title of any section. There are six heading levels <h1> - <h6> which decrease in size from 1 to 6.

The <p> tag represents a paragraph of text. The <a> tag is an inline-text semantic creates a Hyperlink to another webpage, file, or location within the same page. This is an inline text semantic which helps define the meaning or purpose of a word, line or text.

LIST Elements

The ul element, the name for which is an abbreviation of unordered list, is used to group a collection of items together in a list, but in a way that doesn’t suggest an order of precedence or importance. The ul might be used for anything from a simple shopping list to a collection of links.

The li element defines an individual list item and can only appear inside the handful of list-related elements detailed above. Each list item is represented by a bullet (for unordered lists, defined by the ul element) or a number or letter (in the case of ordered lists, defined by the ol element).

The <ul> and <li> tags are used in unison to create an Unordered List of items, which will display as bullet points. You could also use the <ol> tag to create an Ordered List, which will display an ascending counter.

There are two other elements that are used frequently enough to mention here. The <div> and <span> tags are valuable in that they give designers the ability to apply targeted styles to a contained set of content. A <div> is a block-level element that is commonly used to identify large groupings of content, while <span> is an inline-level element commonly used to identify smaller groupings of text within a block-level element.

Allow me to point out two things about the <img /> tag: unlike the other tags used, it is self-closing and it requires src="..." and alt="..." attributes. Attributes are useful for manipulating the state of a particular element or referencing a source as illustrated. Although outside of the scope of our focus here, there are two attribute types that you must become familiar with: class="..." and id="...". Attributes are always declared in HTML, but their functionality is usually declared in CSS and JavaScript. The one caveat is using attributes to make a website accessible and that is covered below.

ATTRIBUTES

HTML Attributes add specificty to elements beyond the scope of styles by adding values that configure the elements or adjust their behavior. There are Dozens of Attributes that can be used for different purposes from simple page layout to building forms or media elements. The main attributes that are found on every website include a href="...", img src="..." alt="", html lang="en-US", xxx title="...", xxx class="..." and xxx id="...".

The title attribute may be used on all elements contained in the body of a document, but is not relevant to the majority of elements found inside the head of the document, e.g. base, script and (perhaps not surprisingly) the title elements.

The id-attribute provides a unique identifies for an element within the document creating a hyperlink. This identifier may also be used in CSS, as a hook that can be used for styling purposes, or with JS, to make changes or add behavior to the element referenced by its unique id. An id-attribute applied to an element must NOT match an id-attribute used elsewhere on the page.

The class-attribute is like an expansion of the id-attribute, but is capable of being assigned to one or more subtypes, which can be used in CSS for styling or JS to make changes and add behavior to elements belonging to a particular subtype.

IMAGES using Alt-attribute provides a text alternative for an important image; applicable with input type. The Longdesc-attribute provides a link to additional information about the image.

The title-attribute provides a title for an element as a ‘tooltip’ on visual browsers; useful with a, link and img providing a title for the linked or embedded resource.

The a element’s purpose is simple: it wraps around text, or an image, or both, and refers to another web page, or another section on the same web page; the user can click on the contained text or image. The a element has a number of special attributes, which are detailed below, but the one that you’ll use most of the time—if it’s not the only one you’ll use—is the href attribute. This attribute indicates the link’s destination, be that another web page, a section of the same web page, or some other type of document, such as an image, a spreadsheet, or a PDF document.

SYNTAX The elements in a web page are contained in a tree structure in which html is the root element that splits into the head and body elements. An element may contain other nested elements (for example, a p element can contain span, etc.). Where this occurs, the opening and closing tags must be symmetrical. If an opening paragraph tag is followed by the opening em element, the closing tags must appear in the reverse order.

Accessibility

The concept of Accessibility focuses on making your web content easy for a wide range of users to access. This may include people with vision impairments, people with mobility problems, and those with cognitive issues.

In web design, Accessibility refers to the practice of designing digital content and applications that can be used by people with visual, motor, auditory, speech or cognitive disabilities. Besides making a site more accessible for users with limitations, research proves that doing so improves SEO, increases download speeds and has better usability.[12] The WC3 and A11Y Project are two major advocates for web accessibility and provide guidelines and patterns for developers to help make their applications accessible.

Here is a short list of Accesibility Criteria that every application should include:

  1. Add Contrasting Color
  2. Design usable :Focus States
  3. Use Lables with Form Fields & Inputs
  4. Write Alternative Text for Images
  5. Use Proper Markup for Content
  6. Support Keyboard Navigation

The Color-Contrast Ration between text and its background should be at least 3/4.5 to 1 depending on the font boldness. The WebAIM group offers a Color Contrast Checker that allows developers to calculate the score for conformance levels in real time. When using color as an indicator, be sure to pair it with an icon, a label or pattern. Charts and graphs are often difficult to read so using shapes and varying sizes can help enhance monochromatic vision. You can use the Color Oracle to simulate what vision impaired folk see in real-time.

Adding a visual cue such as an underline text style will help an element stand out. Focus Indicators help people know which element has the keyboard focus and helps with navigating a website. Elements that should be focusable are links, form fields, widgets, buttons and menu items. Be sure to create a state that is highly visible, with good contrast to stand out from the rest.

People who use screen readers navigate form using the tab key and the <label> element is equipped to handle this action. Never use placeholder text for accessibility because it will be passed over during tabbing. There are plenty of resources for creating Accessible Forms.

Vision impaired users rely on screen readers to 'hear' what is being displayed and the <alt=""> attribute is used to convert text to speech. Simply describe the image or what is happening in the background.

Web browsers read the content and the type of HTML element that contains it. Browsers create their own structure of how the page renders using the elements. Using inline-styles will negatively affect accessibility.

Navigation is accessible only when the <nav> element is properly markedup; this makes it easy to navigate a page through interactive elements that include links, buttons and input fields. Add a :focus state to add a visual indicator. Navigation should be intuitive, minimal and flow from left to right, top to bottom. Test your website using the tab key.

When you think your code is up to par and accessible, download the Accessibility Engines web browser extension to Check a Websites Accessibility.
