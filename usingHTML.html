What is HTML?

HTML is a markup language that defines the structure of the content on a webpage. Every HTML document is a hierarchical structure made up of Elements that dictate how content is displayed and/or sectioned. HTML Elements are used by Internet browsers to read and display HTML content to a user. Elements usually have an opening <element> and closing </element> component, but there are a few elements that are self-closing.

Information contained within any <element> can be configured through the use of Attributes, which adjust its behavior in a manner that meets a specific criteria. An attribute is placed in the opening component of an element, followed by a delimiter and its actual value, like so: attribute="value". There are more than 100 attributes that can be paired with elements depending on the objective. To outline conceivable uses would be a tall order as attributes are declared in HTML, but their functionality is declared in CSS and JavaScript. Obviously, these topics go well beyond the scope of this lesson with the one caveat being using attributes to improve website accessibility. Moving forward I will introduce attributes where they can be expected to be used, but I now present the most common.

The href="..." element specifies the URL of a linked resource while the src="..." element specifies the URL of embedded content. The rel element specifies the relationship of the target object to the link object while. The alt="..." attribute provides an alternative textual description for an image or any HTML object.

The title="..." attribute provides a title for an element as a tooltip in web browsers. It is most useful for embedded or linked resources. Although it may be used on all elements contained in the body of a document, it is not relevant to the majority of elements found inside the head of the document. Do not confuse with the <title> element.

The class="..." attribute is used in CSS for styling or JavaScript to make changes and add behavior to elements belonging to a particular subtype. It can be assigned to one or more subtypes unlike the id="..." attribute, which provides a unique identifier for an element within the document to create a hyperlink. This identifier may also be used in CSS and JavaScript in the same fashion as the class attribute with one exception: an id-attribute applied to an element must NOT match an id-attribute used elsewhere on the page.

With that out of the way I would like to shift attention to requirements for developing an HTML template.

Required Elements for an HTML Template

When creating an HTML template, seven is the magic number as it takes as many <element>s to structure a functional document. The illustration below highlights what is required, followed by a brief explanation for each.

The Main Root

The <!DOCTYPE html> is the first line of a page; it declares that this is an HTML file. The <html> element is on the second and final line of a page. This element is the outermost container that acts as a wrapper for everything on the entire page. There are only two direct descendants of this element and they are explained below.

The Head

The <head> element is the area that contains all meta information about a particular page, instructing the browser where to find style sheets while defining relationships that the document has to others in the web site. It is a direct descedant of the <html> element.

The first line in this area is the <meta> element, which is a self-closing tag used to provide meta data about a webpage, its author, etc. Making a website mobile compatible, for example, is one way to use this tag and will require three attributes:

charset="utf-8"
Declares the character encoding of the page or script.
name="viewport"
A value used by the server for identifying its usage.
content="width=device-width, initial-scale=1.0"
A value associated with name (or http-equiv).

The second line in this area is the <title> element, which is used to document the webpage in the browser. Whatever is placed here will be visible on the browser tab. The third line in this area is the <link> element, which specifies the resources incorporated into a website and will always contain two specific attributes:

rel="stylesheet"
Specifies the relationship to the link object.
href="stylesheet"
The URL of a linked resource (internal or external).

Web browsers use the combination of this information to coordinate all resources required to make the website function as it was designed, however, absolutely none of this information is visible to the user. Once this technical aspect is complete, I shift my focus to structuring content layout.

The Body

The <body> element wraps all content that will be displayed on screen (or in other media, such as print), such as headings, paragraphs, images, tables, and so on. It is also a direct descedant of the <html> element, but it is in this area that we developers create magic for our users!

Defining Sections

During the planning phase, I often imagine how I want the content to be displayed physically and how that content should behave. Doing so helps me determine which HTML element to use. When defining page sections, I know that I can rely on the Header, Nav, Article, Section, Main, Aside, Div and Footer elements to get me started.

The <header> element is the first section users encounter and contains introdutory information related to the website such as a logo, author or navigational aids. The <nav> element allows users to navigate a website, providing links within the current document or to an external source. This element can be used within a <header> or independently, and should be reserved for primary navigation sections only including global navigation, a table of contents, previous/next links, etc.

The <main> element is not an element that should be used to section content as it represents content that is directly related to the central topic of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document, or the central functionality of an application. This element is essentially the main container within the <body> element, possessing all information outside of content that is repeated across a set of documents or document sections such as sidebars, navigation links, copyright information, site logos or search forms.

The <article> element represents a self-contained composition in a document that is intended to be distributable such as a forum post, a newspaper article, or a blog entry. This is 1 of 4 elements that I use to section main areas of content. The <section> element represents a standalone section contained within an HTML document. It can be nested inside of the <article> element to create distinguishable sections.

The <aside> element is used for content that is indrectly related to the documents main content such as sidebars, inserts, or brief explanations. These elements are frequently presented as sidebars or call-out boxes. This element can be nested within other section elements and act as an identifier container for content related to that specific secion. The <div> element is a generic block-level container used to identify large groupings and the flow of content. It has no effect on the content or layout until styled using CSS, which is conducted when employing a class"..." or id"..." attribute.

The <footer> element typically contains information about the author of the section, copyright data or links to related documents. It is placed at the bottom of the section root or used at the bottom of any sectioning elements.

Defining Content

Content elements are used to organize sections of content and are nested within any sectional element. There are dozens of elements that can be used to markup content depending on whether you want to use a sectional heading, add text, create a form, include some sort of multimedia or script, add interactivity and much more. At this juncture, I definitely want to add some context for how elements should be used to define content. In this regard, I will outline components for a new magazine to demonstrate the power of HTML.

Navigation

The first goal that I target when creating a website is building an efficient navigation system that will allow users to seemlessly traverse the site. This project will include three separate pages consisting of the home page, the about page and a linke to an external page. I can complete this goal by using an Unordered List with List Items that contain Anchors. Allow me to explain.

The <ul> element represents an unordered list of items typically rendered as a bulleted list and is used for grouping a collection of items that do not have a numerical ordering precedence. It is nested within the <nav> section and acts as a container for individual list-items or <li> elements. For every list-item placed within an unordered list, an anchor element must be used. The <a> element controls the hyperlink that is created when the href="..." attribute is employed. The file name included within this attribute has the ability to transport the user to another page, a location within the same page, a file, an email address or any other URL browser supported protocol handlers. The example below outlines the hiearchy of a navigation and includes the optional target="_blank" attribute that is used to open that an external link in another browser tab.

<nav class="mainNav">
  <ul>
     <li>
        <a href="index.html">Home</a>
        <a href="about.html">About</a>
        <a href="URL.com" target="_blank">URL</a>
     </li>
  </ul>
</nav>

Before concluding this section, I would like to make a note regarding attributes in relation to the <link> element used in the <head> section. The required rel="..." attribute is used to specify the relationship between target and link objects. There are two values that I use for such: "canonical" (helps webmasters prevent duplicate content issues by specifying the "preferred" version of a page as part of SEO)[*] and "stylesheet" (defines an external resource to be used as a stylesheet).

Understanding rel="..." attribute linking-types is a necessary evil that I must further investigate in order to improve my hand when creating <link>s. Another attribute vital to this element is the type="..." attribute, which helps clarify the content type and will contain a value of "text/css" by default when not included. It is most useful when linking a media source, in which case it would also be wise to include a sizes="H×W" attribute to control its height and width. The title="..." attribute will create a tooltip which is a visual cue for the resource. The media="..." attribute is peculiarly important depending on ones approach to leveraging the @media(query)s of CSS "stylesheet"s. Inclusion of this attribute will load said stylesheet only when a specific media condition is true.
A media query is a CSS concept that I will expand upon during the next course.

Learn about the Ordered List

The <ol> an ordered list which is used to render a numbered list that can be modified through use of attributes. It can be nested within a <ul> and accepts two attributes, start and type, to modify how its items are numbered. The start="..." integer attribute specifies the start value for numbering individual list items. For example, I would use <ol start="4"> to start numbering elements from the #4 and this would mark the first <li> as such. The type="..." attribute indicates the numbering type with 'a', 'A', 'i', 'I' and '1' representing lowercase and uppercase letters, lowercase and uppercase Roman numerals; and numbers (default), respectively.

Text Content

Now I want to include a title, and some paragraph text. I could use the <h1-h6> and the <p> elements for the title and paragraph, respectively, so I will explore these elements first.

The <h1-h6> elements represent section headings that can be used as section titles. Best practice mandates that the <h1> be used only once per page and to avoid skipping levels, which frustrates screen readers. Headings may be nested as subsections to reflect the organization of the content of the page. Most screen readers can generate an ordered list of headings on a page, which can help a person quickly determine the hierarchy of the content. [*] If I wanted to create a multi-level heading for the top news story, I could use the <hgroup> element for such a case. This element allows the primary heading for a document section to be grouped with any secondary headings.

With a heading in place, I can focus on the content that I want to add to the story. The <p> element will allow me to do just that, creating perfectly aligned blocks of text. I could validate the story I am writing by using the <blockquote> element to provide a personal quote or to reference a source of factual information.

<article class="...">
    <hgroup>
        <h1>H1 = Main Title</h1>
        <h2>H2 = Secondary-Title</h2>
    </hgroup>
    <p>blah blah blahblah blah blahhhhh... blah</p>
</article>

In the old days, the <table> element was the go-to element for structuring tabular data. Nowadays, thanks to CSS flexbox and grid properties, I never use this element.

Inline Text Semantics

As was illustrated, the <a> element is used to transport a user to another source, but it is generally used to wrap around text and/or an image providing interaction that can provide not only a link, but also an image, spreedsheet or PDF document. This element is but one of many semantic inline text elements, which are used to define the meaning, structure, or style of a word, line, or any arbitrary piece of text.

Some inline text elements are handy for styling text. The <b>/<strong>, <em>/<i> and <u> elements will make the text bold, italic or underlined, respectively. The <mark> element is useful for highlighting or notating important text. The <code> element can be used to style text in a monospace fashion.

Other inline text elements control the sizing of text. The <small> element will decrease the text size and is useful for a copyright or things of that nature. The <sub> and <sup> elements specify that inline text be displayed as subscript or superscript.

And there are quite a few inline text elements that provide additional meaning to text. The <span> element is an inline-level element commonly used to identify smaller groupings of text within a block-level element. The <cite> element is used to describe a reference to a cited creative work; it must include either the title or the URL of that work and most likely will employ the <q> element, which indicates that the enclosed text is a short inline quotation. The <abbr> element represents an abbreviation or acronym; an optional title attribute can provide a description in the form of a tooltip. If present, the title attribute must contain this full description and nothing else. You can also use the <hr> element to create a stylish line-break between sections of content.

Images

The <img /> element embeds an image into a document. It requires a src="..." attribute for the image file path whose extension will take .svg, .jpg, or .png formats. Although optional, a description of the image will enhance accessibility for screenreaders and this can be accomplished using the alt="..." attribute. There are other attributes that can be applied for styling images, notably, crossorigin and srcset. I prefer using CSS properties (ie border, object-position, margin, width) to control image styling. Moreover, the <img /> element pairs well with a <figure> element, which represents a self-contained container that pairs well with the captioning <figcaption> element.

Forms

The <form> element represents a document section that contains interactive controls for submitting information to a web server. It requires two attributes: action="..." and method="...". The former is URI of a program that processes the form information while the latter is the HTTP method that the browser uses to submit the form. The method attribute will accept either "post" or "get" as its value. Every form that I have ever made used the "post" value, which corresponds to the HTTP POST method whereas form data is included in the body of the form and sent to the server.

The <fieldset> element is used to logically group form element controls within a web form. The <legend> element captions its parent (<fieldset>) like a title for a paragraph. The <label> element share similarities with the <fieldset> in that it also provides a caption for its parent. Its importance rests in how, when used, it links a form control to the associated descriptive text in an unambiguous way that is useful for screen readers and that this element can be applied to any item in a user interface. The <input> element fits the bill in this regard as it is used to create interactive controls for web-based forms in order to accept data from users. What is most fascinating is how the <label> and <input> elements work in unison via their associated attributes. There are two ways to associate these attributes: implicitly nest the <input> attribute directly inside of the <label> element or apply id="..." and for="..." attributes, both with the same value, to the <input> and <label> elements.

The semantics of the <input> element varies considerably depending on the value of its type="..." attribute. Aside from the attributes available to use for the <input> element, the type="..." attribute itself offers 34 possible values to choose from. When this attribute is not specified it will provide a default "text"type.

I want to point out two other attributes to complete the list of that which is required of the <form> element. The value="..." attribute is a control value that is required when the elements type="..." value is "radio" or "checkbox". The name="..." attribute is the name of the control, which is submitted with the control's value as part of the form data. It is required to be used with the <input> element.

The <textarea> element represents a multi-line plain-text editing control, useful for users to input information to be relayed in an email or for providing a comment on a blog. There are five friendly attributes that can be applied, starting with the id="..." which respectively associates itself with the <label> element for accessibility purposes so long as the values are identical. The name="..." attribute is used to set the name of the associated data point submitted to the server when the form is submitted. The row="..." and cols="..." attributes specify the precise size for the <textarea> while the maxlength="..." attribute sets the maximum number of characters that the <textarea> is allowed to contain. This element also accepts several attributes, the most useful of which are autocomplete, autofocus, disabled, readonly and required.

Interactive Elements

The <details> element creates a disclosure widget in which information is visible only when the widget is toggled. Use the self-contained open attribute to programmatically keep open the widge. A summary, caption or legend can be applied to the widget by employing the <summary> element .

The <dialog> element is an experimental technology that represents a dialog box or other interactive component, such as an inspector or window. Dialogs are hidden by default, but can be made active and available for interaction by replacing the id="..." with the open attribute in the <dialog> element. <form> elements can be integrated within a dialog by specifying them with the attribute method="dialog". When such a form is submitted, the dialog is closed with a returnValue attribute set to the value of the submit button used. -- The ::backdrop CSS pseudo-element (experimental) can be used to style behind a <dialog> element, for example to dim inaccessible content whilst a modal dialog is active. The backdrop is only drawn when the dialog element is displayed with dialog.showModal(). The <script> element is not an interactive element. Making a Page Accessible

The concept of Accessibility focuses on making your web content easy for a wide range of users to access. This may include people with vision impairments, people with mobility problems, and those with cognitive issues.

In web design, Accessibility refers to the practice of designing digital content and applications that can be used by people with visual, motor, auditory, speech or cognitive disabilities. Besides making a site more accessible for users with limitations, research proves that doing so improves SEO, increases download speeds and has better usability.[12] The WC3 and A11Y Project are two major advocates for web accessibility and provide guidelines and patterns for developers to help make their applications accessible.

Here is a short list of Accesibility Criteria that every application should include:

  1. Add Contrasting Color
  2. Design usable :Focus States
  3. Use Lables with Form Fields & Inputs
  4. Write Alternative Text for Images
  5. Use Proper Markup for Content
  6. Support Keyboard Navigation

The Color-Contrast Ration between text and its background should be at least 3/4.5 to 1 depending on the font boldness. The WebAIM group offers a Color Contrast Checker that allows developers to calculate the score for conformance levels in real time. When using color as an indicator, be sure to pair it with an icon, a label or pattern. Charts and graphs are often difficult to read so using shapes and varying sizes can help enhance monochromatic vision. You can use the Color Oracle to simulate what vision impaired folk see in real-time.

Adding a visual cue such as an underline text style will help an element stand out. Focus Indicators help people know which element has the keyboard focus and helps with navigating a website. Elements that should be focusable are links, form fields, widgets, buttons and menu items. Be sure to create a state that is highly visible, with good contrast to stand out from the rest.

People who use screen readers navigate form using the tab key and the <label> element is equipped to handle this action. Never use placeholder text for accessibility because it will be passed over during tabbing. There are plenty of resources for creating Accessible Forms.

Vision impaired users rely on screen readers to 'hear' what is being displayed and the <alt=""> attribute is used to convert text to speech. Simply describe the image or what is happening in the background.

Web browsers read the content and the type of HTML element that contains it. Browsers create their own structure of how the page renders using the elements. Using inline-styles will negatively affect accessibility.

Navigation is accessible only when the <nav> element is properly markedup; this makes it easy to navigate a page through interactive elements that include links, buttons and input fields. Add a :focus state to add a visual indicator. Navigation should be intuitive, minimal and flow from left to right, top to bottom. Test your website using the tab key.

When you think your code is up to par and accessible, download the Accessibility Engines web browser extension to Check a Websites Accessibility.

Using Unicode

In HTML, sometimes writing is not as simple as it seems. For example, when I want to write a symbol I cannot simply type it into my keyboard; I must manually input its Character Reference. Character references are better known as Unicode, which is a best practice and International Standard ISO/ICE 10646 Universal Coded Character Set (UCS). In HTML, such references are known as an Entity, which are used to display reserved characters that would otherwise be interpreted as HTML code in the browser. Entities can be created using a character entity reference, but in my studies I have become accustomed to using numeric character references. The syntax for this element begins with an ampersand & and ends with a semicolon ; like so: &#nn;. In this example, the nn represents the numeric value of the symbol that I want to include.
