Every HTML document is a hierarchical structure made up of elements and their content. This section of the reference describes all the different types of element that you can use to write an HTML document.

The html element is the outer container for everything that appears in an HTML document. It can only contain two elements as direct descendants, namely the head element and a body element. It can also contain comments. Although the html element is the outermost element, it does not contain the DOCTYPE—the doctype must come before the html element. This element will have a place on every single web page you ever create!

The head element is the wrapper for all the head elements that, collectively, instruct the browser where to find style sheets, or define relationships that the document has to others in the web site (link); provide meta information (meta); and include scripts that the document will need to apply later on (script). The title elementis the only head element that is absolutely required.

The body element wraps around all of the content that will be displayed on screen (or in other media, such as print), such as headings, paragraphs, images, tables, and so on. Use body to hold all the document content that you want to be visible to the reader.

The div element is a generic block-level element. It doesn’t convey any meaning about its contents (unlike a p element that signifies a paragraph, or an h1 element that would indicate a level 1 or level 2 heading, respectively); as such, it’s easy to customize it to your needs. The div element is currently the most common method for identifying the structural sections of a document and for laying out a web page using CSS. The div element can be used to group almost any elements together —it can contain any inline or block-level elements you choose, so it has no typical content.

HEAD / LINK Elements

The elements listed in this section are all contained within the head element, and either provide extra information about the page, or reference other resources that are required for the page to display or behave correctly.

Link Creates links between a document and a stylesheet (CSS); no-limit to usage of #Links.
link href="url" media="media types" rel="relationship" type="content type"
link rel="stylesheet" href="basic.css"/
*The charset attribute is intended to identify the character set used in the document that’s referenced within the link element.
*The href defines the location of the resource.
*The rel attribute defines the relationship of referenced document to the referencing document, the rev attribute defines the relationship that the referenced document classification.
*The type is an optional attribute of the element, that is used to identify the MIME type of the linked resource. Although it’s optional, browsers may not load the resource if its missing or incorrect.
The a element’s purpose is simple: it wraps around text, or an image, or both, and refers to another web page, or another section on the same web page; the user can click on the contained text or image. The a element has a number of special attributes, which are detailed below, but the one that you’ll use most of the time—if it’s not the only one you’ll use—is the href attribute. This attribute indicates the link’s destination, be that another web page, a section of the same web page, or some other type of document, such as an image, a spreadsheet, or a PDF document.

Meta Provides information about the document content. meta content="string" { http-equiv="http response header" | name="string" } / The following meta tags are the most relevant in terms of search engine indexing and interpretation: meta name="description" content="A brief description of page content to appear in SEO results" meta name="keywords" content="reference, SitePoint, HTML, XHTML, standards" Script Used to enclose a series of statements in a scripting language (JS) or refer to an stylesheet (CSS) through the ‘src’ attribute, saved in JS. **Most Valuable for STYLING!**
*n this example, the src attribute refers to a script file named "complex.js": script type="text/javascript" src="/scripts/complex.js
*The language attribute tells the browser which scripting language is to be used inside the script block (or in the file referred to by the src attribute, as shown in the example). It may be the name of the scripting language, or a version-specific implementation of the language (see the values below). The appropriate way to identify the contents of the script element is through the type attribute (for example, you might specify the content as "text/javascript"), and use JavaScript object/feature detection techniques within the script rather than assume that all browsers which support a given version of a scripting language support all of the features of that language—no questions asked.

STYLE Used to override the styles set in a linked style sheet, like so:
link rel="stylesheet" href="basic.css" /*which contains the selector h2 {color:#066;}*/
style type="text/css">h2 {color:#000;}style /* override color set in main.css for this page only*/

TITLE used to summarize the content or purpose of the document.

LIST Elements

The ul element, the name for which is an abbreviation of unordered list, is used to group a collection of items together in a list, but in a way that doesn’t suggest an order of precedence or importance. The ul might be used for anything from a simple shopping list to a collection of links.

The li element defines an individual list item and can only appear inside the handful of list-related elements detailed above. Each list item is represented by a bullet (for unordered lists, defined by the ul element) or a number or letter (in the case of ordered lists, defined by the ol element).

SYNTAX The elements in a web page are contained in a tree structure in which html is the root element that splits into the head and body elements. An element may contain other nested elements (for example, a p element can contain span, etc.). Where this occurs, the opening and closing tags must be symmetrical. If an opening paragraph tag is followed by the opening em element, the closing tags must appear in the reverse order.

The concept of Accessibility focuses on making your web content easy for a wide range of users to access. This may include people with vision impairments, people with mobility problems, and those with cognitive issues.

FORMS using Fieldset and Legend Elements to logically group related form controls, and provide a title for the grouping via the Legend Element. The Label element links a form control to the associated descriptive text in an unambiguous way (useful for non-visual browsers, using screen readers).

IMAGES using Alt-attribute provides a text alternative for an important image; applicable with input type. The Longdesc-attribute provides a link to additional information about the image.

ATTRIBUTES

HTML Attributes add specificty to elements beyond the scope of styles by adding values that configure the elements or adjust their behavior. There are Dozens of Attributes that can be used for different purposes from simple page layout to building forms or media elements. The main attributes that are found on every website include a href="...", img src="..." alt="", html lang="en-US", xxx title="...", xxx class="..." and xxx id="...".

The title attribute may be used on all elements contained in the body of a document, but is not relevant to the majority of elements found inside the head of the document, e.g. base, script and (perhaps not surprisingly) the title elements.

The id-attribute provides a unique identifies for an element within the document creating a hyperlink. This identifier may also be used in CSS, as a hook that can be used for styling purposes, or with JS, to make changes or add behavior to the element referenced by its unique id. An id-attribute applied to an element must NOT match an id-attribute used elsewhere on the page.

The class-attribute is like an expansion of the id-attribute, but is capable of being assigned to one or more subtypes, which can be used in CSS for styling or JS to make changes and add behavior to elements belonging to a particular subtype.
